# 🛒 Shopping List Database

Projeto desenvolvido para praticar conceitos fundamentais de **MySQL**, com foco na criação, modificação e gerenciamento de tabelas utilizando comandos DDL (*Data Definition Language*) e DML (*Data Manipulation Language*).

## 🎯 Objetivo

O principal objetivo deste projeto é demonstrar meu aprendizado em SQL por meio da construção gradual de uma tabela de lista de compras.

Durante o desenvolvimento, utilizei diversos comandos de alteração de estrutura para simular a evolução de um banco de dados e praticar operações comuns realizadas por desenvolvedores e administradores de banco de dados.

## 📚 Conceitos praticados

- Criação de banco de dados (`CREATE DATABASE`)
- Criação de tabelas (`CREATE TABLE`)
- Inserção de registros (`INSERT INTO`)
- Alteração de colunas (`ALTER TABLE`)
- Definição de valores padrão (`DEFAULT`)
- Restrições de unicidade (`UNIQUE`)
- Chave primária (`PRIMARY KEY`)
- Incremento automático (`AUTO_INCREMENT`)
- Modificação de tipos de dados (`MODIFY`)
- Renomeação de colunas (`CHANGE`)
- Remoção de colunas (`DROP COLUMN`)
- Exclusão de tabelas (`DROP TABLE`)
- Consulta de dados (`SELECT`)
- Visualização da estrutura da tabela (`DESCRIBE`)

## 🗂️ Estrutura da tabela

A tabela armazena informações de uma lista de compras, incluindo:

| Campo | Descrição |
|---------|------------|
| id | Identificador único do produto |
| product | Nome do produto |
| quantity | Quantidade do item |
| weight | Peso ou unidade informada |
| price | Preço do produto |
| shopping_date | Data da compra |

## ⚠️ Observação

Este projeto foi desenvolvido com finalidade educacional.

Algumas alterações foram realizadas utilizando múltiplos comandos `ALTER TABLE` de forma intencional para praticar diferentes operações de modificação da estrutura de um banco de dados. Em um ambiente de produção, parte dessas definições normalmente seria criada diretamente na primeira versão da tabela.
